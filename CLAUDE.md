## Architecture

Provide a high‑level overview of the system architecture:

- **Components**:
  - **Postgres** – central relational database used by Airflow metadata and Metabase.
  - **pgAdmin** – web UI to manage the Postgres instance.
  - **Zookeeper & Kafka** – streaming platform for ingesting and buffering data.
  - **Kafdrop** – UI to inspect Kafka topics and messages.
  - **MinIO** – S3‑compatible object storage for raw files.
  - **Airflow** (init, webserver, scheduler) – orchestrates ETL jobs, interacts with Kafka/MinIO and runs dbt transformations.
  - **Metabase** – analytics/dashboard service reading from Postgres.
- **Data Flow**: Data Source → **Kafka** (via Zookeeper) → **MinIO** (object store) → **Airflow** (orchestrates dbt jobs) → **Postgres** (metadata & analytical store) → **Metabase** (visualisation). Optional UI tools (pgAdmin, Kafdrop) monitor intermediate services.
- **Technology Stack**:
  - **Docker & Docker‑Compose** for container orchestration.
  - **Python 3.11** for Airflow DAGs and any custom scripts.
  - **Apache Airflow 2.8.0**, **dbt** for data transformation.
  - **PostgreSQL 15**, **MinIO**, **Kafka 7.4**.
  - **Metabase** for BI dashboards.
  - Supporting tools: **pgAdmin**, **Kafdrop**.
